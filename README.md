# 🛡️ FraudShield AI — Real-Time & Dataset Fraud Detection System

A startup-grade fintech web app that detects fraudulent transactions in real
time and analyzes entire datasets in seconds. Every verdict comes from a
**trained machine-learning model** (RandomForest) — there are **no hardcoded or
fake predictions**.

> ⚡ Instant single-transaction detection · 📁 Bulk CSV analysis · 📉 Risk
> scoring (LOW / MEDIUM / HIGH) · 📊 Live glassmorphism dashboard with charts.

---

## ✨ Features

- **Real-Time Single Detection** — score one transaction (amount, location,
  device, time) and get `prediction`, `risk_score` (0–100%), and `risk_level`.
- **Dataset-Level Analysis** — upload a CSV; the model scores every row and
  returns totals, fraud %, average risk, and the top 5 highest-risk records.
- **Live Dashboard** — animated count-up analytics cards, **Line chart**
  (Hourly Traffic & Fraud), **Donut chart** (Verdict Distribution), recent
  transactions table, all auto-refreshing every 4 seconds.
- **Risk Scoring** — `model.predict_proba()` → risk score → level:
  `0–40% LOW 🟢 · 40–70% MEDIUM 🟡 · 70–100% HIGH 🔴`.
- **Persistence** — every single + dataset prediction is stored in SQLite and
  feeds the global stats.
- **Bonus** — fraud-spike alert banner, recent-transactions table, and
  **Download Report (CSV)**.
- **High-end UI** — futuristic fintech theme: animated gradient background,
  glass cards, neon glow borders, hover animations, skeleton/loader states,
  fully responsive.

## 🤖 Machine Learning

The model is a scikit-learn **Pipeline** so preprocessing and the classifier are
saved together in `model.pkl` (no duplicated logic between train and serve):

1. **Preprocessing**
   - Numeric (`amount`, `time`): median imputation + `StandardScaler`.
   - Categorical (`location`, `device`): most-frequent imputation +
     `OneHotEncoder(handle_unknown="ignore")`.
2. **Model**: `RandomForestClassifier` (`class_weight="balanced"`).
3. **Prediction**: `model.predict()` + `model.predict_proba()`; probability ×100
   becomes the risk score.

> The included `data/dataset.csv` is **synthetic** (generated by
> `data/generate_dataset.py`) but the fraud label is produced from a latent risk
> function with realistic correlations + noise, so the RandomForest learns
> genuine patterns. To use your own data, replace `dataset.csv` (columns:
> `amount, location, device, time, is_fraud`) and re-run training.

## ⚙️ API

| Method | Endpoint            | Description                                              |
| ------ | ------------------- | ------------------------------------------------------- |
| POST   | `/predict`          | Score one transaction → prediction, risk_score, level   |
| POST   | `/analyze-dataset`  | Upload CSV → totals, fraud %, avg risk, top-5 high-risk  |
| GET    | `/stats`            | total_processed, fraud_detected, normal_count, avg_risk |
| GET    | `/transactions`     | Full transaction history (`?limit=N` optional)          |
| GET    | `/hourly`           | Hourly total vs fraud (for the line chart)              |
| GET    | `/meta`             | Known locations/devices + model metrics                 |
| GET    | `/report`           | Download full history as CSV                            |

## 🚀 Setup & Run

```bash
# 1. Clone + enter
git clone https://github.com/harimalasb-code/fraudshield-ai.git
cd fraudshield-ai

# 2. Create a virtual environment + install deps
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt

# 3. (Optional) regenerate data + retrain the model
python data/generate_dataset.py
python model/train.py

# 4. Run the backend (also serves the frontend)
cd backend
uvicorn main:app --reload --port 8000
```

Then open:

- **App UI**: http://localhost:8000/app/index.html
- **API docs**: http://localhost:8000/docs

> The frontend is served by FastAPI at `/app`, so the API and UI share an
> origin. You can also open the `frontend/*.html` files directly — they fall
> back to `http://localhost:8000` for API calls.

### Try it

1. **Single detection** — go to **Detect**, enter e.g. `amount=8500,
   location=Lagos, device=ATM, time=2` → expect a **HIGH risk / Fraud** verdict.
2. **Dataset analysis** — on the **Dashboard**, upload
   `data/sample_transactions.csv` and watch the summary, charts, and top-5 table.
